========================================
4. QUY TRÌNH HOÀN THÀNH 1 MODULE
========================================
STEP_1: Tạo nhánh feature từ main
    - git checkout main
    - git pull
    - git checkout -b feature/<module-name>

STEP_2: Phát triển chức năng + giao diện của module trên nhánh feature
    - Commit thường xuyên, message nên kèm issue Redmine nếu có:
      "Implement cart add item (refs #12)"

STEP_3: Test chéo trên nhánh feature
    - Thành viên khác checkout nhánh feature để test.
    - Lỗi được ghi trên Redmine:
        + Tracker: Bug
        + Assigned to: DEV của module.
        + Target version: v0.x.0 tương ứng.

STEP_4: Fix bug trực tiếp trên nhánh feature
    - Commit dạng:
      "Fix total price rounding (fixes #21)"
    - Đảm bảo các bug mức Blocker/Critical của module đều đã xử lý.

STEP_5: Merge module vào main
    - Tạo Pull Request: feature/<module-name> -> main
    - Review, test lại, rồi merge vào main.

STEP_6: Gắn tag phiên bản trên main
    - Ví dụ:
        + v0.1.0 : xong MODULE_1 (Auth & Identity)
        + v0.2.0 : xong MODULE_2 (Category)
        + v0.3.0 : xong MODULE_3 (Product & Catalog)
        + v0.4.0 : xong MODULE_4 (Shopping Cart)
        + v1.0.0 : xong MODULE_5 (Order & Payment), hệ thống chạy ổn định.
    - Lệnh:
        git checkout main
        git pull
        git tag v0.3.0 -m "Hoàn thành module Product & Catalog (đã test chéo và fix bug)"
        git push origin v0.3.0
